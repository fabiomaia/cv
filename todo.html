Todo Lists

Most software engineers are used to leaving TODO comments in their source code, e.g., 
  • process.env.NODE_TLS_REJECT_UNAUTHORIZED = '0' // TODO: this is cheating

  • process.env.NODE_TLS_REJECT_UNAUTHORIZED = '0' // TODO: this is cheating
What many don't realize is that if you follow this methodology of keeping If you do this in a certain way you can even effectively version control many of your tasks. Then you can produce a list of todos quite easily: 
#!/bin/sh
grep -RInH "TODO" .
  • -R
  • -I
  • -n
  • -H
For many cases (e.g., small to medium projects or solo projects) this is actually quite productive and really simple. It works anywhere grep is installed. Feel free to wrap it in a utility script that you can extend as you go 
#!/bin/sh
DIR="$1"
STR="TODO"
grep -RInH "$STR" "$DIR"
